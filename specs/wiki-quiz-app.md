# WikiBlank — Wikipedia穴埋めクイズアプリ 設計書

## コンセプト
Wikipediaの記事から自動生成される穴埋めクイズアプリ。好きなページを選んで、難易度を選んでスタート。Wikipediaが存在する限りクイズは無限に増え続ける。

## 競合分析

| アプリ | 特徴 | 弱点 |
|--------|------|------|
| Wiki穴埋めクイズ（2022） | プロ野球・坂道アイドル限定 | ジャンル固定、更新されない |
| Redactle | Wikipedia記事の単語を推測 | 穴埋めではなく推測ゲーム。Web専用 |
| WikiMaster | Wikipedia記事から4択クイズ | 穴埋めではない |
| QuizKnock Wikipedia | 目次から記事を当てる | Webのみ、単発企画 |

**WikiBlankの差別化:**
- 全Wikipedia記事が対象（日本語・英語）
- 穴埋め形式（読んで覚える学習効果が高い）
- 難易度で穴の量が変わる（初心者〜マニアまで）
- タイムアタック＋ランキングで競争要素
- 人気ページランキングでトレンド発見

## 画面構成

### 1. ホーム画面
- 検索バー（Wikipediaページ検索）
- おすすめページ（今日のピックアップ、人気ランキング）
- カテゴリ一覧（歴史、科学、地理、スポーツ、エンタメ、etc.）
- 自分の成績サマリー（正解率、連続記録）

### 2. ページ選択画面
- Wikipedia記事のプレビュー（冒頭3行）
- 難易度選択：
  - **かんたん** — キーワードの10%を穴埋め（固有名詞中心）
  - **ふつう** — キーワードの25%を穴埋め（固有名詞＋数字）
  - **むずかしい** — キーワードの50%を穴埋め（固有名詞＋数字＋重要語）
  - **えげつない** — 70%以上を穴埋め
- 「スタート」ボタン

### 3. クイズ画面
- Wikipedia記事本文が表示される（穴埋め部分は [ ___ ] で表示）
- 穴埋め箇所をタップ → キーボードで入力
- 正解 → 緑で表示、不正解 → 赤で表示して正解を薄く表示
- 画面上部にタイマー表示
- ヒントボタン（1文字目を表示。使うとスコア減）
- 「ギブアップ」→ 残りの答えを全表示

### 4. リザルト画面
- 正解率（○/○問）
- クリアタイム
- スコア（正解数 × 難易度倍率 - ヒント使用ペナルティ）
- ランキング内での順位（このページ内）
- 「もう一回」「別のページ」「シェア」ボタン

### 5. ランキング画面
- **ページ別ランキング**: 各Wikipedia記事ごとのスコアTop10
- **人気ページ**: 今週プレイ回数が多い記事
- **全問正解タイム**: 難易度別の最速クリア
- **ユーザーランキング**: 累計スコアTop100

### 6. マイページ
- プレイ履歴
- 得意カテゴリ（カテゴリ別正解率）
- バッジ・実績（初クリア、連続正解、全問正解 etc.）
- 設定（言語切替、通知）

## 穴埋めロジック（AIコスト不要）

### 穴埋め対象の選定（テキスト処理のみ）
1. Wikipedia APIで記事本文を取得
2. テキストを文単位に分割
3. 品詞・パターンで穴埋め候補を抽出：
   - **固有名詞**: 大文字始まり（英語）、カタカナ語（日本語）
   - **数字**: 年号、人数、距離、金額
   - **重要語**: 記事タイトルに含まれる単語、太字の単語、リンクされている単語
4. 難易度に応じて穴埋め数を調整

### 正解判定
- 完全一致（表記ゆれは正規化して比較）
- 数字は半角/全角どちらもOK
- カタカナ/ひらがなは両方受け付け

## 技術構成

### クライアント（iOS先行）
- **言語**: Swift / SwiftUI
- **アーキテクチャ**: MVVM
- **ローカルDB**: SwiftData（プレイ履歴、キャッシュ）

### サーバーサイド
- **Wikipedia API**: 記事取得（無料、レートリミットあり）
  - `https://ja.wikipedia.org/api/rest_v1/page/summary/{title}` — 概要
  - `https://ja.wikipedia.org/api/rest_v1/page/html/{title}` — 本文HTML
  - `https://ja.wikipedia.org/w/api.php?action=parse` — パース済みテキスト
- **バックエンド**: Firebase（認証、ランキング、人気ページ集計）
  - Firestore: ランキングデータ、ユーザープロフィール
  - Firebase Auth: 匿名認証→アカウント連携
  - Cloud Functions: ランキング集計、人気ページ更新

### コスト構造
- **Wikipedia API**: 無料
- **Firebase**: 無料枠で相当カバー可能（Spark Plan）
  - Firestore: 読み取り50,000/日、書き込み20,000/日
  - ユーザー1万人程度までは無料枠内
- **AI/LLMコスト**: なし（テキスト処理のみ）
- **Apple Developer**: $99/年

## マネタイズ

### フリーミアム
- **無料**: 1日5ページまでプレイ可能、広告あり
- **プレミアム（月額¥300 or ¥2,500/年）**:
  - 無制限プレイ
  - 広告なし
  - 詳細ランキング
  - カスタム難易度
  - プレイ履歴の統計分析

### 広告
- クイズ終了後のリザルト画面にインタースティシャル（5回に1回）
- ホーム画面下部にバナー
